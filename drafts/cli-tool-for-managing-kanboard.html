CLI Tool for Managing Kanboard

Posted on Thu 18 May 2017 in articles

My team at work uses Kanboard to manage our OpenStack sites and their progression from procuring hardware, configuring hardware, and deploying OpenStack on them. This probably isn't the most common use for kanban tools, but it mostly works in our case. One of the coolest features of Kanboard is that it has a pretty thorough API for handling all sorts of actions. Frequently, we have need to make mass modifications of tasks and projects on Kanboard. This typically happens when we update a template project, from which many other projects (like 50+) were created. Someone could go about updating those projects one at a time, but this is 2017. Hitting the API is pretty easy, but to really speed things up, I needed to come up with some tool that iterate through many projects/tasks/subtasks/etc. and update them all at once. So, I set out to create a command-line tool to handle this.

The Setup

Fortunately, there is an API client for Python that prevents us from having to handle writing curl statements via subprocess calls. And installing this module is as simple as pip install kanboard. Next, I need to figure out how I'm going to handle the command-line aspect of the tool. I could stick with good 'ol argparse but how I envision this tool would make writing it with argparse a pain in the ass.
